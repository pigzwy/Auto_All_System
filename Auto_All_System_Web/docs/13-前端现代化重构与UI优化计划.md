# 前端现代化重构与 UI 优化计划

## 1. 背景与目标

目前系统前端采用传统的 Element Plus 默认样式，视觉风格较为陈旧，缺乏 SaaS 产品的专业感和现代感。
本计划旨在通过引入 **Tailwind CSS** 和 **现代化 UI 设计规范**，将系统重构为清爽、高效、具有科技感的管理平台。

### 1.1 核心目标
*   **视觉升级**: 从“传统后台”转向“现代化 SaaS”。
*   **体验优化**: 增加微交互、过渡动画和响应式适配。
*   **开发提效**: 使用 Tailwind CSS 原子类减少手写 CSS/SCSS，统一设计语言。

## 2. 技术栈与规范

*   **核心框架**: Vue 3 + TypeScript + Vite
*   **UI 组件库**: Element Plus (用于复杂交互组件，如 Table, Form, Dialog)
*   **样式引擎**: Tailwind CSS v3.4 (用于布局、间距、颜色、排版)
*   **图标库**: Element Plus Icons

### 2.1 设计语言规范 (Design System)

#### 配色方案 (Color Palette)
*   **主色调 (Primary)**: 蓝色系 (Blue-600) - 用于按钮、高亮、链接。
*   **背景色 (Background)**:
    *   页面背景: `bg-gray-50` (极淡灰，减少视觉疲劳)
    *   卡片背景: `bg-white` (纯白，配合阴影)
    *   侧边栏: `bg-slate-900` (深蓝黑，强调导航)
*   **文字色 (Typography)**:
    *   标题: `text-gray-800`
    *   正文: `text-gray-600`
    *   辅助/说明: `text-gray-400`

#### 阴影与圆角
*   **卡片**: `rounded-xl shadow-sm border border-gray-100 hover:shadow-md`
*   **按钮**: `rounded-lg`

## 4. 后续执行计划 (Roadmap)

### Phase 2: 核心业务页面改造 (进行中)

目标：将列表页从简单的“表格堆砌”改为“筛选+卡片/表格”的复合布局。

#### 4.1 专区管理 (Zones)
- **现状**: 简单的 Table 列表。
- **改造计划**:
    - 头部增加“状态概览”统计条。
    - 列表项支持“卡片模式”和“表格模式”切换。
    - 专区详情页改为左右分栏布局（左侧信息，右侧操作日志）。

#### 4.2 虚拟卡管理 (Cards)
- **现状**: 密集的表格数据。
- **改造计划**:
    - 重点优化卡片状态标签（正常、冻结、过期）的视觉样式。
    - 卡号、CVV 等敏感信息增加“点击显示/复制”交互。
    - 充值/提现操作改为侧边抽屉 (Drawer) 而非弹窗 (Modal)，体验更流畅。

#### 4.3 个人中心 (Profile)
- **现状**: 简单的表单。
- **改造计划**:
    - 改为由多个“设置卡片”组成的网格布局（基本信息、安全设置、API Key）。
    - 增加头像上传预览和交互。

## 5. 开发指南

在后续开发中，请遵循以下原则：

1.  **优先 Tailwind**: 能用 Tailwind 类名解决的，不要写 `<style scoped>`。
2.  **语义化布局**: 使用 `<header>`, `<main>`, `<aside>`, `<section>` 等语义标签。
3.  **保持一致**: 所有的卡片、按钮、输入框圆角和阴影必须遵循 `2.1` 中的规范。
4.  **图标使用**: 统一使用 Element Plus Icons，注意大小和颜色协调。
