# 快速开始指南

本指南将帮助您在 5 分钟内开始使用比特浏览器自动化管理工具。

---

## 📋 准备工作

### 1. 检查系统要求

- ✅ Windows 10/11 操作系统
- ✅ Python 3.8 或更高版本
- ✅ 比特浏览器已安装

### 2. 验证 Python 安装

打开命令提示符（CMD）或 PowerShell，运行：

```bash
python --version
```

应该显示 `Python 3.8.x` 或更高版本。

---

## 🚀 安装步骤

### Step 1: 安装 Python 依赖

```bash
# 进入项目目录
cd Auto_All_System_Pyqt

# 安装依赖包
pip install -r data/requirements.txt

# 安装 Playwright 浏览器
playwright install chromium
```

**安装时间**: 约 3-5 分钟

---

### Step 2: 配置比特浏览器

#### 2.1 启动比特浏览器

双击启动比特浏览器应用程序。

#### 2.2 启用 API 接口

1. 打开比特浏览器设置
2. 找到"API设置"或"接口设置"
3. 启用 HTTP API
4. 确认端口为 `54345`（默认）
5. 保存设置

**验证 API 是否可用**：

在浏览器中访问：http://127.0.0.1:54345/api/v1/browser/list

如果返回 JSON 数据，说明 API 已成功启用。

---

### Step 3: 准备数据文件

在 `data/` 目录下创建以下文件：

#### 3.1 accounts.txt（账号文件）

**格式**：`email----password----recovery_email----2fa_secret`

**示例**：
```
user1@gmail.com----MyPassword123----backup1@gmail.com----JBSWY3DPEHPK3PXP
user2@gmail.com----MyPassword456----backup2@gmail.com----HXDMVJECJJWSRB3H
```

**说明**：
- 用 `----` 分隔四个字段
- 每行一个账号
- 2FA 密钥是 TOTP secret（16位字符）

#### 3.2 proxies.txt（代理文件）

**格式**：`proxy_type://username:password@host:port`

**示例**：
```
http://user1:pass1@proxy1.example.com:8080
socks5://user2:pass2@proxy2.example.com:1080
```

**说明**：
- 支持 http、https、socks5 代理
- 每行一个代理
- 如果代理不需要认证，格式为：`http://proxy.com:8080`

#### 3.3 cards.txt（卡片文件）

**格式**：`card_number|exp_month|exp_year|cvv`

**示例**：
```
4111111111111111|12|2025|123
5555555555554444|06|2026|456
```

**说明**：
- 用 `|` 分隔四个字段
- 卡号不要有空格
- 月份用两位数（01-12）
- 年份用四位数

## 💡 快速体验核心功能

### 功能 1：批量创建浏览器窗口

1. 在主界面找到"创建窗口"选项卡
2. 输入模板窗口ID（可选，留空则使用默认配置）
3. 点击"批量创建窗口"
4. 系统会自动：
   - 读取 accounts.txt 中的账号
   - 读取 proxies.txt 中的代理
   - 为每个账号创建一个浏览器窗口
   - 自动配置代理和账号信息

**预期结果**：在右侧日志区看到创建进度

---

### 功能 2：SheerID 验证

1. 切换到"SheerID验证"选项卡
2. 输入 SheerID API Key
3. 导入链接文件（sheerIDlink.txt）
4. 点击"开始批量验证"
5. 系统会自动提交验证并轮询状态

**格式示例（sheerIDlink.txt）**：
```
user1@gmail.com----https://verify.sheerid.com/...
user2@gmail.com----https://verify.sheerid.com/...
```

---

### 功能 3：一键全自动处理

这是最强大的功能，整合了所有步骤。

1. 切换到"一键全自动"选项卡
2. 配置参数：
   - **SheerID API Key**: 输入你的 API Key
   - **并发线程数**: 建议 3（初次使用）
   - **一卡几绑**: 一张卡绑定几个账号（如：3）
   - **延迟设置**: 各步骤之间的等待时间
3. 选择要处理的账号（勾选复选框）
4. 点击"开始全自动处理"

**处理流程**：
```
登录账号 → 检测资格 → 提取链接 → SheerID验证 → 绑卡订阅
```

**处理时间**：每个账号约 3-5 分钟

## 📊 查看结果

### 1. 实时日志

在应用程序右侧的日志区域，可以看到实时处理进度和详细日志。

### 2. 账号状态

在账号列表中，每个账号会显示当前状态：
- 🔵 **pending** - 待处理
- 🟡 **processing** - 处理中
- 🟢 **verified** - 已验证
- 🟣 **subscribed** - 已订阅
- 🔴 **failed** - 失败

### 3. 数据库

处理结果会自动保存到 `data/accounts.db`。
使用 SQLite 工具可以查看详细数据。

### 4. 文本文件

根据处理结果，账号会被分类保存到：
- `已验证未绑卡.txt` - 已通过 SheerID 验证但未绑卡
- `已绑卡号.txt` - 成功绑卡订阅
- `无资格号.txt` - 不符合学生优惠资格
- `超时或其他错误.txt` - 处理失败

## 🔧 配置优化

### 推荐配置（新手）

```
并发线程数: 3
一卡几绑: 3
登录后等待: 5秒
获取链接后等待: 3秒
验证后等待: 5秒
绑卡后等待: 10秒
```

### 高级配置（有经验）

```
并发线程数: 5-8
一卡几绑: 根据需要调整
延迟时间: 可适当减少
```

**注意**：过多并发可能导致被检测为异常行为。

## ⚠️ 常见错误

### 错误 1: 无法连接比特浏览器

**症状**：`Connection refused` 或 `无法连接到 API`

**解决方案**：
1. 确认比特浏览器已启动
2. 检查 API 端口是否为 54345
3. 在浏览器访问 http://127.0.0.1:54345/api/v1/browser/list 验证

---

### 错误 2: Playwright 浏览器未安装

**症状**：`Browser not found` 或 `Executable doesn't exist`

**解决方案**：
```bash
playwright install chromium
```

---

### 错误 3: 模块导入失败

**症状**：`ModuleNotFoundError: No module named 'xxx'`

**解决方案**：
```bash
pip install -r data/requirements.txt
```

---

### 错误 4: 文件编码错误

**症状**：`UnicodeDecodeError` 或乱码

**解决方案**：
- 确保所有文本文件使用 UTF-8 编码
- 使用 Notepad++ 或 VS Code 打开文件
- 选择"转换为 UTF-8 编码"并保存

## 💡 使用技巧

### 技巧 1：批量处理前先测试

在处理大量账号前，先选择 1-2 个账号测试，确认流程正常。

### 技巧 2：合理设置延迟

- 网络较慢时，增加延迟时间
- 网络良好时，可适当减少延迟

### 技巧 3：定期备份数据

```bash
# 备份数据库
copy data\accounts.db data\accounts_backup.db

# 备份账号文件
copy data\accounts.txt data\accounts_backup.txt
```

### 技巧 4：使用日志排查问题

如果某个账号失败，查看日志中的详细错误信息，对症下药。
